# GKE MCP v0.10.0 — Kubernetes 運用を MCP で自動化する最前線

GKE MCP が v0.10.0 (2026-03-23) をリリースした。v0.9.0 からわずか数日で、4つの新スキルと Docker イメージサポートが追加された。Go バージョンは v1.25.6 → v1.25.8 に更新、golangci-lint が CI 統合された。Kubernetes 運用の自動化がどこまで来たのか見ていく。

## 新スキル一覧

### Golden Image Finder（v0.10.0 新規）

コンテナイメージの**ゴールデンイメージ**（組織が承認したベースイメージ）を検索するスキル。**セキュリティチーム**が承認したイメージを AI エージェントが自動的に選択できるようになる。

### ComputeClasses 作成（v0.10.0 新規）

GKE の **ComputeClass** を作成するスキル。ComputeClass はワークロードに適したマシンタイプの優先順位を定義する抽象化レイヤーで、**GPU ワークロード**や高メモリジョブの配置を最適化する。

### Workload Scaling（v0.10.0 新規）

**ワークロードスケーリング**管理スキル。HPA/VPA の設定や、負荷に応じたスケーリング戦略の提案を AI エージェントが行える。

### Workload Security（v0.9.0、2026-03-23 以前）

前バージョンで追加されたセキュリティスキル。**Pod Security Standards** の適用状況確認やセキュリティコンテキストの推奨設定を提供する。

## 運用自動化の4象限

この4スキルを整理すると、GKE 運用の主要な関心事がカバーされていることがわかる。

```
          Security           Performance
         +------------------+------------------+
  Build  | Golden Image     | ComputeClasses   |
         | Finder           |                  |
         +------------------+------------------+
  Run    | Workload         | Workload         |
         | Security         | Scaling          |
         +------------------+------------------+
```

Legend / 凡例:
- Security: セキュリティ
- Performance: パフォーマンス
- Build: ビルド時
- Run: ランタイム

ビルド時のセキュリティ（イメージ選択）、ビルド時のパフォーマンス（コンピュートクラス）、ランタイムのセキュリティ（Pod Security）、ランタイムのパフォーマンス（スケーリング）。

## Docker イメージとクラスタ作成ベストプラクティス

v0.10.0 (2026-03-23) では **Docker イメージ**も提供されるようになった (#183)。これにより、GKE MCP サーバー自体を GKE 上で動かすというメタ的な構成が可能になる。

また、v0.9.0 で GKE クラスタ作成の**ベストプラクティスドキュメント**が追加されており (#177)、AI エージェントがクラスタを作成する際のガードレールとして機能する。

### 非推奨 API の置換

v0.10.0 では `ServerOption.HasTools/HasResources` が `Capabilities` に置換された。これは **MCP SDK** の進化に追従する変更で、**後方互換性**への影響がある。SDK レベルで適切に処理されているが、カスタム MCP サーバーを構築している場合は確認が必要だ。

## 技術的な変更

- `ServerOption.HasTools/HasResources` が `Capabilities` に置換された。MCP SDK の進化に追従する**非互換変更**だが、SDK レベルで適切にハンドルされている
- Go **v1.25.8** への更新（v1.25.6 → v1.25.8）
- **golangci-lint** の CI 統合によるコード品質強制化

## まとめ

GKE MCP v0.10.0 (2026-03-23) は、Kubernetes 運用の4つの主要関心事——**イメージセキュリティ**、**コンピュートリソース最適化**、**ランタイムセキュリティ**、**オートスケーリング**——を MCP スキルとして提供する。Docker イメージの提供により、デプロイ自体もコンテナ化された。v0.9.0 のセキュリティスキルから v0.10.0 のスケーリング・ゴールデンイメージまで、Kubernetes 運用の**エージェント自動化**は着実に実用段階に近づいている。
