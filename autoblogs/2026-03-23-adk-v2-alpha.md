# ADK v2.0.0a1 — Google Agent Development Kit の次世代への布石

Google の Agent Development Kit (ADK) Python に v2.0.0a1 アルファタグが付けられた。v1.27.x で機能が充実し続ける中でのアルファリリースは、次世代への布石を示している。3言語（Python/Go/JS）の ADK エコシステム全体の動向を見る。

## ADK Python: v1.27.x の充実

v2 アルファの存在を語る前に、v1.27.x 系の充実ぶりを見ておきたい。

### v1.26.0 の基盤

v1.26.0 で入った Agent Registry と Skills 仕様準拠は、v1.27.0 の機能爆発の基盤となった。Skills のバリデーション、エイリアス、スクリプト、自動インジェクション——これらが揃ったことで、ツールの追加が加速した。Apigee LLM 統合（`/chat/completions` とストリーミング）、User Personas（評価フレームワーク）、OAuth2 `id_token` ネイティブサポート、A2A インターセプターフレームワークも v1.26.0 で追加されている。

## v2.0.0a1 は何を意味するか

v2.0.0a1 のアルファタグは、v1.x の安定性を維持しながら v2 の設計実験を行うためのものだ。main ブランチは v1.27.2 以降も 1111 コミット先行しており、活発な開発が続いている。

v2 で予想される変更：

- **API の整理**: v1.x で `experimental_` プレフィックスだった機能の安定化
- **Skills ファースト設計**: One Skill per Toolset（GenAI Toolbox v0.30.0 と同じ方向）
- **Durable Runtime のコア化**: 現在オプショナルな永続ランタイムのデフォルト化
- **A2A ネイティブ**: エージェント間通信のファーストクラスサポート

## 3言語エコシステムの状況

### ADK Go v0.6.0

ADK Go v0.6.0 の主要変更：

| 変更 | 詳細 |
|------|------|
| Apigee モデルサポート | Apigee モデル追加 (#639) |
| RemoteAgent 拡張ポイント | パーツ変換の拡張ポイント (#627) |
| pathPrefix 設定 | API ランチャーに設定追加 (#616) |

## ADK Python の破壊的変更履歴

| バージョン | 変更 |
|-----------|------|
| v1.24.0 | `credential_manager` が `tool_context` を受け取るよう変更 |
| v1.23.0 | OpenTelemetry BigQuery プラグイントレーシング移行 |

v1.x 系での破壊的変更は最小限に抑えられている。v2 では、蓄積された設計上の負債を一括して解消する機会になるだろう。

## まとめ

ADK Python v2.0.0a1 は、v1.27.x の機能充実と並行して進む次世代設計の実験だ。Skills ファースト、Durable Runtime コア化、A2A ネイティブ——v1.x で育てた機能を v2 で統合する方向が見える。3言語 SDK のうち Python が圧倒的にリードしており、Go/JS は特定のユースケースに集中する分業体制が確立されている。
