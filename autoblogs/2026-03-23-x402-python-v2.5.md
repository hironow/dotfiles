# x402 Python v2.5.0 — Permit2 & Gas Sponsorship で決済プロトコルが実用段階へ

x402 の Python SDK が v2.5.0 をリリースし、Permit2 と Gas Sponsorship 拡張が追加された。Go v2.6.0、TypeScript Core v2.7.0 と並行して進化する x402 エコシステムの現在地を整理する。なお x402 の API バージョン管理は ACP の v2026-01-30 や UCP の v2026-01-23 と異なる日付ベースではなく、セマンティックバージョニングを採用している点が特徴だ。

## x402 とは

x402 は HTTP 402 (Payment Required) ステータスコードを活用するインターネットネイティブ決済プロトコル。API エンドポイントに決済を組み込み、エージェント間の自律的な支払いを可能にする。Coinbase が主導し、オープンソースで開発されている。

## Python v2.5.0 の主要変更

### Permit2 拡張

Uniswap の Permit2 コントラクトを活用した承認メカニズム。従来の ERC-20 approve は各トークン/各コントラクトごとに個別承認が必要だったが、Permit2 では一度の署名で複数トークンの一括管理が可能になる。

Python SDK でこれが利用可能になったことで、サーバーサイドの決済処理でも Permit2 フローが使える。

### Gas Sponsorship 拡張

ガス代（トランザクション手数料）をスポンサーが負担する拡張。エンドユーザーやエージェントが ETH を保持していなくても、ERC-20 トークンだけで決済を完了できる。

これはエージェント間決済の実用化に直結する。AI エージェントに「ETH を充填しておく」運用は現実的ではないが、「USDC だけ持たせる」なら管理可能だ。

## 3言語 SDK の並行進化

| 言語 | 最新バージョン | 注目機能 |
|------|-------------|---------|
| **Go** | v2.6.0 | Permit2 E2E テスト、SVM スキーム、指数バックオフリトライ |
| **Python** | v2.5.0 | Permit2 & Gas Sponsorship 拡張 |
| **TypeScript** | v2.7.0 | Permit2 シミュレーション、ERC-7710、ERC-20 Gas Sponsorship |

TypeScript が先行して機能を実装し、Go と Python が追従するパターンが見える。ERC-7710（delegatable permissions）は TypeScript のみだが、いずれ他言語にも展開されるだろう。

## エコシステムの拡大

x402 のエコシステムパートナーは急速に拡大している。Tollbooth（OSS API ゲートウェイ）、WalletIQ、Laso Finance など、決済インフラから金融サービスまで幅広い。

特に注目すべきは:

- **Tollbooth**: x402 対応のオープンソース API ゲートウェイ。既存の API に x402 決済を後付けできる
- **Stellar サポート**: EVM 以外のブロックチェーンへの拡張
- **Polygon メインネット**: CDP Facilitator での Polygon サポート追加（未リリース）

## Go v2.6.0 の主な変更

Go 版は Python より先行して v2.6.0 をリリースしている。

| 変更 | 詳細 |
|------|------|
| Permit2 E2E テスト修正 | TypeScript サーバーの Permit2 E2E テスト修正 |
| ルート設定バリデーション | 初期化時のルート設定検証 (#1364) |
| 指数バックオフリトライ | `GetSupported` の 429 レスポンスに対するリトライ (#1357) |
| ERC20 Gas Sponsorship | ガススポンサーシップ拡張実装 (#1336) |

## Python v2.4.0 → v2.5.0 の差分

v2.4.0 (MCP トランスポート安定化、camelCase シリアライゼーション修正 #1122、無効 Facilitator レスポンス処理修正 #1506) を経て、v2.5.0 で Permit2 & Gas Sponsorship が追加された。ACP v2026-01-30 が Payment Handlers を導入した 2026-01-30 前後から、決済プロトコル全体で Permit2 対応の機運が高まっている。

## 未リリースの注目変更

- **Signed Offer & Receipt Extension**: オプショナルな署名付きオファー＆レシート。決済の証跡管理を標準化する動き (#935)
- **Express スタイルルートパラメータ**: `:param` 形式の動的ルートマッチング。Express.js ユーザーには馴染み深いパターンで、x402 の導入障壁を下げる (#1313)
- **Polygon メインネット**: CDP Facilitator に Polygon メインネットサポート追加 (#1564)

## まとめ

x402 Python v2.5.0 の Permit2 & Gas Sponsorship は、エージェント間決済の実用化に向けた重要なピースだ。「エージェントに USDC を渡すだけで自律的に支払いができる」世界が近づいている。3言語 SDK（Go v2.6.0、Python v2.5.0、TypeScript v2.7.0）の並行進化と、ACP v2026-01-30（2026-01-30）・UCP v2026-01-23 との相互補完が、エージェント決済エコシステムとしての成熟を示している。
